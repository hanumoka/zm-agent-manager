# 문서 자동 갱신 규칙

작업 중 아래 조건에 해당하면 **관련 문서를 즉시 갱신**한다.
사용자에게 별도 승인을 구하지 않고, 갱신 사실만 간결하게 알린다.

## 자동 갱신 트리거

### 즉시 갱신 (작업 완료 직후)

| 트리거 | 갱신 대상 | 갱신 내용 |
|--------|----------|----------|
| roadmap 체크리스트 항목 완료 | `docs/roadmap/phase-*.md` | `- [ ]` → `- [x]` |
| 새 버그/이슈 발견 | `docs/troubleshooting/known-issues.md` | 이슈 추가 |
| 새 아이디어 언급 | `docs/ideas/INBOX.md` | 아이디어 추가 |
| 정책 변경 합의 | 해당 정책 파일 (`docs/policies/*.md`) | 변경 사항 반영 |
| PRD 변경 합의 | `docs/requirements/PRD.md` | 변경 사항 반영 |
| 새 기술적 결정 | 해당 문서 또는 `CLAUDE.md` | 결정 사항 반영 |

### 조건부 갱신 (변경 규모에 따라)

| 조건 | 행동 |
|------|------|
| 코딩 컨벤션 예외 합의 | `coding-conventions.md`에 예외 사항 추가 |
| 로드맵 순서 변경 합의 | `ROADMAP.md` + phase 파일 업데이트 |
| 새 Phase/Milestone 추가 합의 | 해당 phase 파일 생성 및 ROADMAP 반영 |

## 갱신 알림 형식

문서 갱신 후 작업 결과와 함께 간결하게 보고:

```
📝 문서 갱신: `docs/roadmap/phase-1-mvp.md` — M1 스캐폴딩 항목 3개 완료 체크
```

## 갱신하지 않는 경우

- 사용자가 단순 질문만 한 경우 (코드 설명, 개념 질문 등)
- 탐색/조사만 수행한 경우 (코드 읽기, 분석 등)
- 임시 실험/테스트인 경우 (사용자가 "테스트", "실험" 언급)

## SESSION_LOG 갱신 타이밍

- SESSION_LOG.md는 `/zm-session-end` 스킬로만 갱신한다 (자동 갱신 대상 아님)
- 단, 사용자가 명시적으로 세션 종료를 요청하면 즉시 갱신
