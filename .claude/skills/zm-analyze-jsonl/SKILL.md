# JSONL 세션 데이터 분석

`~/.claude/` 디렉토리의 JSONL 세션 데이터를 분석한다.
**읽기 전용** — 어떤 파일도 수정하지 않는다.

## 분석 대상

인자가 있으면 해당 파일만, 없으면 최근 세션 파일을 자동 선택.

## 분석 항목

1. **레코드 타입 분류**
   - `file-history-snapshot`, `user`, `assistant` 등 타입별 빈도
   - 각 타입의 필드 구조

2. **메시지 체인 분석**
   - `uuid` ↔ `parentUuid` 관계 매핑
   - 대화 깊이(depth) 통계

3. **도구 호출 패턴**
   - `assistant` 응답 내 `tool_use` 블록 구조
   - 사용된 도구 종류 및 빈도

4. **파일 크기 및 라인 수**
   - 파일 크기, 총 라인 수, 평균 라인 크기

## 출력 형식

```
## JSONL 분석 결과

### 파일 정보
- 경로: ...
- 크기: ...MB, 라인: ...개

### 레코드 타입 분포
| 타입 | 건수 | 비율 |
|------|------|------|
| ... | ... | ...% |

### 메시지 체인
- 최대 깊이: ...
- 평균 깊이: ...

### 도구 호출 통계
| 도구 | 호출 수 |
|------|---------|
| ... | ... |

### 타입 정의 제안
(분석 결과를 바탕으로 TypeScript 타입 초안 제시)
```

## 주의사항
- `~/.claude/` 데이터를 **절대 수정하지 않는다**
- 대용량 파일은 head/tail로 샘플링하여 분석
- 민감한 사용자 데이터는 요약만 표시 (원본 내용 출력 최소화)
