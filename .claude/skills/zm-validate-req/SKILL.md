# 요구사항 검증

사용자의 요구사항을 프로젝트 문서 전체와 교차 검증한다.

## 입력

- `$ARGUMENTS`: 검증할 요구사항 텍스트
- `"all"` 입력 시: PRD 내 모든 기능 정의를 정책/로드맵과 전체 정합성 검사

## 검증 절차

### 1. 관련 문서 로드

아래 문서를 모두 읽는다:

- `CLAUDE.md` — 핵심 원칙
- `docs/requirements/PRD.md` — 기능 정의
- `docs/policies/coding-conventions.md` — 코딩 컨벤션
- `docs/policies/git-workflow.md` — Git 규칙
- `docs/policies/doc-management.md` — 문서 관리 규칙
- `docs/roadmap/ROADMAP.md` — 전체 로드맵
- `docs/roadmap/phase-*.md` — Phase별 상세
- `docs/troubleshooting/known-issues.md` — 알려진 이슈
- `docs/ideas/INBOX.md` — 아이디어 목록

### 2. 검증 항목

| 검증 | 설명 |
|------|------|
| **모호성** | 범위, 기준, 조건이 불명확한 부분 |
| **PRD 충돌** | 기존 기능 정의와 모순되는 부분 |
| **정책 위반** | 코딩 컨벤션, Git 규칙, 핵심 원칙 위반 |
| **로드맵 이탈** | 현재 Phase 범위 밖이거나 의존성 위반 |
| **기술 제약** | Electron 아키텍처, 성능, 보안 관점의 제약 |
| **중복** | 이미 정의된 기능이나 진행 중인 작업과 중복 |
| **누락** | 필요하지만 언급되지 않은 사항 (에러 처리, 경계값 등) |

### 3. 출력 형식

```
## 요구사항 검증 보고서

### 검증 대상
> (사용자 요구사항 원문)

### 검증 결과 요약
- ✅ 통과: N건
- ⚠️ 경고: N건
- 🚨 충돌: N건

### 상세

#### ✅ 통과 항목
- PRD 정합성: 통과
- 코딩 컨벤션: 통과
- ...

#### ⚠️ 경고 (진행 가능하나 주의 필요)
1. [경고 제목]
   - 내용: ...
   - 관련 문서: [파일:라인]
   - 권장 조치: ...

#### 🚨 충돌 (해결 필요)
1. [충돌 제목]
   - 요구사항: ...
   - 기존 정책: [파일:라인] — ...
   - 충돌 내용: ...
   - 해결 옵션:
     A) ...
     B) ...

### 최종 판단
(진행 가능 / 수정 후 진행 / 보류)
```

### 4. "all" 모드 (전체 정합성 검사)

PRD의 모든 기능(F1~F10)에 대해:
- 로드맵에 배정되었는지
- Phase 간 의존성이 올바른지
- 정책과 모순이 없는지
- 중복 정의가 없는지

전체 정합성 매트릭스를 출력한다.
